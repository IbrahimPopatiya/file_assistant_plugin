# 🗂️ File Assistant Plugin

A Windows desktop background utility that **watches your folders and active File Explorer window**, logs every file activity (created, modified, deleted, renamed/moved) to a local SQLite database, and lets you **ask an AI (OpenAI GPT-4) questions about your recent file activity** in plain English.

## ✨ Features

- 👀 **Folder watcher** – monitors a fixed directory (`backend/watched/`) for file changes using `watchdog`.
- 🪟 **Active Explorer watcher** – automatically detects whichever folder is currently open in File Explorer (via `pywin32` / `pywinauto`) and watches it live, switching as you navigate.
- 🧾 **Event logging** – every create/modify/delete/move/rename event is timestamped, deduplicated, and stored in a local SQLite database (`file_events.db`).
- 🤖 **AI Q&A over file history** – ask natural-language questions (e.g. "what files were deleted last week?") and get an answer generated by OpenAI's GPT-4, grounded in your logged file events.
- 🔒 **Single-instance lock** – prevents multiple watcher instances from running at once.
- 📜 **History viewer / DB scripts** – simple scripts to inspect logged events directly from SQLite.

## 🛠️ Tech Stack

- **Language:** Python 3.10
- **File system monitoring:** [watchdog](https://pypi.org/project/watchdog/)
- **Windows automation:** [pywin32](https://pypi.org/project/pywin32/), [pywinauto](https://pypi.org/project/pywinauto/)
- **AI:** [OpenAI API](https://pypi.org/project/openai/) (GPT-4)
- **Web framework (API layer):** [FastAPI](https://fastapi.tiangolo.com/) + [Uvicorn](https://www.uvicorn.org/)
- **Database:** SQLite (via Python's built-in `sqlite3`)

## ⚙️ Setup

### Prerequisites
- Windows OS (uses Windows Explorer / COM automation)
- Python 3.10+

### Installation

1. **Clone the repo**
   ```bash
   git clone https://github.com/IbrahimPopatiya/file_assistant_plugin.git
   cd file_assistant_plugin
   ```

2. **Create and activate a virtual environment**
   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Configure your environment**

   Copy the example config and add your own OpenAI API key:
   ```bash
   copy backend\config.example.py backend\config.py
   ```
   Then set your API key as an environment variable (recommended, never commit real keys):
   ```bash
   setx OPENAI_API_KEY "sk-your-key-here"
   ```

## 🚀 Usage

### Start the active folder watcher (main app)
```bash
cd backend
python main.py
```
This will detect the folder currently open in File Explorer and start logging file events to `file_events.db`.

### Start the fixed-directory watcher
```bash
cd backend
python file_watcher.py
```
This watches `backend/watched/` for any file changes.

### Ask the AI about your file activity
```bash
cd backend
python ai_handler.py
```
This fetches recent file events from the database and asks OpenAI a sample question about them. You can extend `ask_openai()` to plug into a chat UI or FastAPI endpoint.

### View logged events
```bash
cd backend
python history_viewer.py
```

## 🔐 Notes on Security

- `backend/config.py` (containing your real API key) is **git-ignored** — never commit it.
- Use `backend/config.example.py` as the template for new setups.
- The SQLite database (`file_events.db`) and log files are also git-ignored since they contain local activity data.
